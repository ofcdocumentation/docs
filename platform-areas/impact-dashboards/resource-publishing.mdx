### About

Resource publishing on the Open Impact Platform provides a structured way to curate, contextualize, and share knowledge within and across organizations. A resource is considered a **container for knowledge**, rather than just a standalone file. It bundles one or more files—such as toolkits, recordings, or research—with an overarching theme, description, and point of contact.

Publishing a resource provides the necessary context and attribution to ensure work can travel responsibly across the network. Organizations can choose to keep resources private for internal coordination, share them within specific Groups, or contribute them to the public **Open Impact Resource Library**—a shared collection of practices and tools intended to support the wellbeing of the planet and people.

**Each Resource space includes:**

- **Contextual information:** Detailed descriptions, related websites, and descriptive tags to make materials discoverable.
- **Multimedia attachments:** Support for PDFs, images, videos, spreadsheets, and external web links.
- **Flexible access control:** Ranging from public visibility to restricted access for specific individuals or groups.
- **Location-based data:** Allowing resources to be tied to specific geographic regions to appear on the platform's map.
- **Attribution and stewardship:** Fields to clearly note usage restrictions, licenses, and responsible organizations.

### Working with Resource Publishing

#### Setting up a Resource - Quick Start

Before publishing, consider who the intended reader is, which files you have ready, and what tags will make the resource most searchable.

\[ Show ‘My Resources’ and ‘\+Add New’ in this screenshot \]

- **To create a resource:** Click **My Resources** in the top menu and select **Add New**.
- **Basic Information:** Enter the title, images, and select the sponsoring organization or group.
- **Visibility & Privacy:** Use the **Description** tab to set your core visibility rules (Public, Private, Connections, or Custom).
- **Attach Content:** Use the **Files** and **Links** tabs to upload new documents or select existing files from your profile.
- **Submit vs. Publish:** There is a critical distinction between these two steps:
  - **Submit:** Finalizes the resource setup and makes the resource card visible where shared.
  - **Publish:** Happens via the **Publishing Wizard** and controls whether the attached files are actually accessible to users.

### Resource Setup Tabs

#### Basic Info Tab

This is where the initial identity of the resource is formed.

- **Title and Tags:** Choose descriptive tags (skills, purpose, or knowledge) to ensure the resource is discoverable by others.
- **Sponsor:** Designate which organization or group is responsible for the resource.

#### Description Tab

The Description tab is the most critical step for controlling how your resource is shared.

- **Resource Library Visibility:** Selecting **"Show in Resource Library"** determines if the resource is submitted for review to appear in the public community collection.
  - _Tip:_ You can set this to "No" while keeping privacy "Public" to share a resource selectively via link without listing it in the public library.
- **Privacy Settings (⚙️Icon):** Fine-tune who can access the resource and its files. Options include **Public**, **Private**, **Connections**, or **Custom** (specific groups or individuals).
- **Note:** Sharing is additive; for example, you can share a resource with a specific group while also keeping it accessible via a public link.

#### Contact Info Tab

This tab manages the geographic and contact context of your resource.

- **Location:** Enter a physical address and click **"get from address."** Note that **Country is a required field**.
- **Privacy:** Use the **"Use map"** option to set a pin manually or enter coordinates, allowing for map-based discovery without disclosing a full street address.

#### Files & Links Tab

This tab serves as the engine for your resource's content.

- **Attachments:** You can drag and drop new files, select existing ones, or use the **Spreadsheet** and **Web Docs** tabs to create new attachments directly. Every resource must have at least one file attached to be published.
- **Links:** Add external URLs (like YouTube or web docs) with descriptive titles.
- **File Reuse:** Files live in your central library and can be attached to multiple resources. Any update made to a file's name or content will automatically reflect everywhere it is used.

### The Publishing Wizard

After clicking **Submit**, you must finalize the process using the Publishing Wizard to release the files.

1. **Open the Wizard:** Click the **gear icon** on the resource banner and select **Publish**.
2. **Project Quality:** Leave this set to **Standard**.
3. **Project Questions:**
   - Keep **"Attach Resources"** selected.
   - Deselect **"In-kind value"**.
   - Deselect **"Additional Notes"**.
4. **Publishing Files:** Check **"Publish"** next to each file you want to be accessible.

### Roles & Permissions

- **Authors & Sponsors:** Responsible for setting visibility, managing attribution, and ensuring content is appropriate for reuse.
- **File Roles vs. Privacy:** In most cases, leave the **file role** set to **Public** during publishing. File roles do not override resource privacy; the resource's privacy settings will determine who can actually see the files.
- **OFC Resource Library Review:** Resources marked for the public library are reviewed by the Open Future Coalition team based on relevance to focus areas including:
  - Regenerative Agriculture & Food Systems
  - Ecological Restoration
  - Community Wellness
  - Infrastructure & Built Environment
- **Modifying Content:** To add or remove files after a resource has been submitted, you must go to the final tab of the authoring workflow and select **"Reopen"**. Once modifications are complete, you must **submit and publish again**.
