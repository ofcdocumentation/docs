# About

The dashboard tool allows users to create visual reports that draw from data collected across forms, workflows, and project activities. Dashboards can combine multiple types of content—including charts, tables, images, videos, and narrative text—to provide a comprehensive view of project progress and outcomes.

Dashboards are created using a drag-and-drop interface that allows users to configure visualizations and organize information in ways that best reflect the needs of their program or audience. Data can be displayed in real time as new information is submitted through program workflows or field-level reporting.

Dashboards can be created at the level of a single project or aggregated across multiple projects and sites, allowing organizations to analyze performance and outcomes across distributed initiatives.

This flexibility supports a range of uses, from internal program monitoring to external reporting and impact storytelling.

Key Features include: 

- **Drag-and-Drop Interface**: Easily arrange content modules for custom dashboard layouts.
- **Multiple Data Sources**: Connect live data (API, project data, web documents) or historical data (CSV, spreadsheets).
- **Rich Content Modules**: Images, videos, links, cards, image carousels, and more.
- **Customizable Charts**: Select from a wide range of chart types, including bar, line, funnel, pie, scatter, and radar charts.
- **Flexible Layouts**: Choose from various column configurations, text sections, and dividers for structured content presentation.
- **Interactive & Shareable**: Publish dashboards, share links, and embed content on external platforms

## Component & Module Breakdown

The Dashboard Publishing Tool is built around **modules**, which are individual content blocks that serve specific purposes for displaying data or visuals.

## Content Modules

- **Attachment:** Use this to upload files, documents, or embed external links, including YouTube and Vimeo videos.
- **Image Carousel:** Showcase multiple images in a scrollable, interactive format.
- **Card:** Create structured informational blocks featuring a title, description, and preview image.

## Data Modules

- **Measurable Results:** Showcase quantitative results by connecting to a Measurable Results Reporting Form for auto-syncing live data, or enter data manually.
- **Charts:** Visualize data using various types:
  - **Area/Line Charts:** Best for trends over time.
  - **Bar Charts:** Best for comparing values across categories.
  - **Pie/Donut Charts:** Best for displaying proportions.
  - **Radar/Scatter Charts:** Best for multi-variable comparisons and correlation analysis.

## Connecting Modules to Data

Modules like Charts and Measurable Results require a data source to function. You can connect data in three primary ways:

1. **Project Data:** Select **Fetch from Project List** to source data from one or more existing projects on the platform. You can then select specific forms and "data keys" (fields) to display.
2. **Spreadsheets:** Select **Fetch from Spreadsheet List** to use spreadsheets already created or uploaded to the platform.
3. **Historical/Manual Data:** Use **Manual Spreadsheet** to type data directly into the editor, or **Manual Data (Comma-separated)** to paste CSV data.
4. **Remote API:** For advanced integrations, select **Fetch from Remote API**. Users should contact [**support@openfuturecoalition.org**](mailto:support@openfuturecoalition.org) for assistance with API setup.

# Privacy & Publishing

## Dashboard Privacy Settings

Privacy settings control who can see the dashboard and where it can be shared. To edit these, click the **gear icon** at the bottom of the edit menu.

- **Public:** Anyone with the link can view the dashboard, even without a platform login.
- **Private:** Visibility is restricted to you, your organization, and the projects it is shared to.
- **Connections:** Visible to all your direct connections.
- **Custom:** Allows for specific sharing rules with certain groups or individuals.

## Publishing to a Project

To display a dashboard within a specific project's "Dashboard" tab:

1. Navigate to the project and go to its **Dashboard** tab.
2. Select your created page from the dropdown menu.
3. Click **Edit selected page** (this opens a new tab), then click **Update Page** to finalize the connection.

# Roles & Permissions

- **Creators:** Only the individual or organization that created the dashboard can access the drag-and-drop editor to modify the layout or data sources.
- **Support:** For technical assistance or API setup, contact [**support@openfuturecoalition.org**](mailto:support@openfuturecoalition.org).
