Groups serve as hubs for collective work — connecting projects and resources under a shared community umbrella.

## Group Projects

### Creating a Group Project

1. Navigate to **Group → Projects**
2. Click **\+ Create Project**
3. Follow the standard [project creation process](/platform-areas/projects/quick-start)
4. The project will automatically be affiliated with the group

### Linking Existing Projects

Already have a project? Affiliate it with your group:

1. Go to **Project → Settings → Affiliations**
2. Search for and select your group
3. The group admin may need to approve the affiliation

### Featuring Projects

Highlight your most important projects on the group homepage:

1. Go to **Group → Settings → Featured Content**
2. Add projects to the featured list
3. Drag to reorder

## Group Resources

### Publishing a Group Resource

1. Navigate to **Group → Resources**
2. Click **\+ Publish Resource**
3. Follow the standard [resource publishing process](/platform-areas/resource-publishing/quick-start)
4. The resource will be attributed to the group

### Resource Collections

Organize related resources into collections for easier discovery:

1. Go to **Group → Resources → Collections**
2. Click **\+ New Collection**
3. Name the collection and add a description
4. Add resources to the collection

### Curating External Resources

Groups can also curate and recommend resources published by others:

1. Find a resource on the platform
2. Click **Recommend to Group**
3. Select your group
4. The resource appears in your group’s recommended list

## Organizing Group Content

### Content Hub

The group’s **Content Hub** tab shows a unified view of:

- Active and recent projects
- Published and recommended resources
- Featured content from members

### Permissions

Control who can add content to your group:

| Action | Default Role Required |
| --- | --- |
| Create project | Member |
| Publish resource | Member |
| Feature content | Admin |
| Curate collections | Moderator |

Customize these defaults in **Group → Settings → Content Permissions**.
