The project file library is a shared workspace for all project documents, assets, and resources.

## Overview

Every project has a built-in file library where members can store and access shared documents. Files in the project library are accessible to all project members according to the project’s visibility settings.

## Uploading Files

### Single File Upload

1. Navigate to **Project → Files**
2. Click **Upload File**
3. Select a file from your device
4. Add a title, description, and tags
5. Click **Upload**

### Bulk Upload

1. In the **Files** tab, drag and drop multiple files into the upload area
2. Files will upload simultaneously
3. Add details to each file after upload

### Supported File Types

See [File Management](/platform-fundamentals/file-management) for a full list of supported file types and size limits.

## Organizing Files

### Creating Folders

1. Click **\+ New Folder**
2. Name the folder
3. Drag files into it, or upload directly to the folder

### Recommended Folder Structure

For most projects, a simple structure works well:

```text
Project Files/
├── Background & Research/
├── Working Documents/
├── Meetings & Notes/
├── Deliverables/
└── Archive/
```

## Version Control

The platform automatically maintains version history for uploaded files.

### Viewing File History

1. Click on a file to open its detail view
2. Click **Version History**
3. Browse previous versions

### Restoring a Previous Version

1. In **Version History**, find the version you want to restore
2. Click **Restore This Version**
3. The previous version becomes the current version (the current version is saved to history)

## File Permissions

By default, all project members can view all files. You can restrict specific files:

1. Open the file detail view
2. Click **Settings → Permissions**
3. Choose who can view or download the file

## Sharing Files Outside the Project

To share a file with someone who isn’t a project member:

1. Open the file
2. Click **Share**
3. Copy the share link
4. Set the link expiration (optional)

<Warning>
  Share links give access to the file to anyone with the link. Use expiring links for sensitive documents.
</Warning>
