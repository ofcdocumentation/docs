### About
Projects on the Open Impact Platform serve as structured environments for coordinating time-bound initiatives, collaborative workstreams, and measurable impact efforts. While Groups organize people around a shared context, Projects provide the operational layer where teams plan, implement, and track specific initiatives, deliverables, and outcomes.

Projects are best suited for efforts that are time-bound, task-focused, and require measurable results. Groups may sponsor one or more Projects, allowing organizations to move from general coordination to specific execution while maintaining visibility and accountability.

**Each Project space includes:**
*   **Shared newsfeed** for timely updates, announcements, and real-time coordination.
*   **Discussion spaces** that allow collaborators to engage in searchable, taggable topic-based dialogue to maintain continuity.
*   **Shared calendaring** to coordinate meetings, milestones, and events throughout the work lifecycle.
*   **Integrated project management tools** to organize workflows, assign tasks, and track progress.
*   **Visual management tools** including Gantt charts, timelines, and swimlanes to monitor project health.
*   **Data collection and dashboards** to gather qualitative and quantitative inputs and visualize impact in real time.

### Working with Projects
#### Setting up a Project - Quick Start
Before setting up your project, consider if the effort is time-bound, what specific measurable results need tracking, and which collaborators require task assignments.

[ Show ‘My Projects’ and ‘+Add new project’ in this screenshot ]

*   **To create a project:** Click the **My Projects** tab in the top menu and select **Create Project**.
*   **Managing Organization:** Select the organization that will govern the project; they will be able to manage tasks, discussions, and calendar invites.
*   **Sponsoring Group:** You must be a creator or manager of a group to select it as a sponsor; the project will then appear on that group's profile.
*   **Location:** Use the **Addresses** field to set a project location; selecting "use map" or "get from address" ensures the project appears on the public map.
*   **Tracking Results:** In the **Preferences** tab, choose the measurable results and status tags you will use to manage tasks.

### Project Tabs
#### Feed Tab
The Feed is the primary hub for real-time coordination. It displays a running chronological set of posts, calendar events, and attachments associated with the project.
*   Allows members to create posts with images and links or comment on existing updates.
*   Includes a "Pending Approvals" tab for Organization profiles to review submitted results and tasks.

#### Discussion Tab
This tab is a long-term repository for ongoing dialogue and knowledge organization.
*   **Create:** Creators or managers can click **Add New** and add tags to make the thread searchable by purpose or skills.
*   **Notifications:** Participants receive email and platform notifications when a new topic is created.
*   **Management:** Managers can moderate threads and delete comments.

#### Files Tab
The Files tab serves as a central repository for all media uploaded to the project.
*   Files appear here automatically after being attached to tasks, feed posts, or discussions.
*   **Visibility:** Files inherit the permissions of the project "container" by default.
*   **Note:** This tab is view-only within the project space; to retroactively change privacy settings or delete a file globally, you must use the Files tab on your individual profile.

#### Management Tab
This tab contains the operational project management tools unique to Project spaces.
*   **Add Task:** Select a task template (e.g., General Task Library) and enter descriptions and priority levels.
*   **Assigning:** You can choose between **Single Assign** (one task for multiple people) or **Bulk Assign** (clones the task for each individual).
*   **Importing:** You can bulk import tasks by copying and pasting data from a spreadsheet into the platform's importer.

#### Visual Management Tools
Tasks are automatically populated into visual views found in the Management tab.
*   **Gantt Chart:** Ideal for visualizing task sequences and durations; users can click and drag bars to update dates.
*   **Swimlanes:** Visualizes task cards in vertical columns based on status; users can drag and drop cards to update progress.

### Roles & Permissions
*   **Creator/Admin:** The primary administrator who is the only one able to edit high-level settings like descriptions and profile images.
*   **Project Managers:** Designated users who can create/assign tasks, manage discussions, and send calendar invites. They must first be added to the Organizational Manager list before they can be assigned at the project level.
*   **Collaborators:** Members who participate in the workspace and can be assigned tasks. They must be a "Connection" of the creator before they can be added to the project via the **Preferences** tab.
